# Architecture Spec Format

This document describes how the architecture spec should be formatted.

The spec should start with an Overview section that provides a high level view of the entire architecture and the motivation. Any fundamental axioms should be spelled out as well.

The remaining sections should describe in sufficient detail the components of the architecture so an implementation plan can be created from the architecture documents.

In order to avoid context overload, the overview and table of contents should be contained in the architecture.md file while other components should each be contained in their own markdown files prefixed with a chapter number and name.

Do not use emojis.
